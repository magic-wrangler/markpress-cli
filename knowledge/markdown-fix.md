# Markdown 修复规则（Markpress 准则）

> **用途**：供 `mpr ai` 知识库、`修复` / `检查` 指令、转换前规则预处理、AI 深度修复共用。  
> **维护**：直接编辑本文件；`mpr ai` **新开对话**后自动生效。实现代码见 `lib/markdown-preprocess.ts`。

## 目标

让 Markdown 能被 **marked** 正确解析并转为 HTML，同时**不改写正文含义**。

常见根因：AI 或工具导出时**过度转义**（多余 `\`）、ATX 标题 `#` 后缺空格。

---

## 处理流水线（两层）

```
读取 .md
  → ① 规则预处理（默认开启，免费、即时）
  → ② AI 深度修复（可选，仅「修复」/ `--ai-fix` 且已配置 API）
  → ③ 规则再跑一遍（AI 之后补全 # 空格、\*\* 等）
  → 有变更则写回源 .md（可用 --no-write-md 关闭）
  → marked 解析 → 套主题 → HTML
```

| 层级 | 触发 | 写回 md |
|------|------|---------|
| ①+② | 转换 / @ / **修复** | **默认写回**（有变更时）；`--no-write-md` 可关闭 |
| **检查** | 仅预览 diff | 确认后才写入 |

关闭规则层：`MARKPRESS_MD_NO_PREPROCESS=1` 或 `convert --no-preprocess`

---

## 用户如何触发修复

### 对话内（推荐）

| 指令 | 模式 | 说明 |
|------|------|------|
| **修复**、**帮我修复**、**请修复**、**修复文档/格式** | fix | 规则 + AI（已配置时）→ diff → 确认写入 |
| **检查**、**帮我检查**、**检查md格式**、**看看文档** | check | **仅规则**，预览 diff → 确认写入 |
| **帮我转换**、**@** | 转换前修复 + **有变更时写回 md** + 出 HTML |
| `fix md` / `check markdown` | fix / check | 英文别名 |

**修复 vs 检查**：

- **检查**：不调 AI，适合快速看 `\` / `#` 问题
- **修复**：有 `DEEPSEEK_API_KEY` 或 `~/.markpress/ai-config.json` 时会走 AI；未配置则等同加强版规则修复

**交互细节**：

- 当前目录仅 1 个 md 时自动选中
- 修复/检查**覆盖写回原文件**，写入前必须确认
- diff 高亮变更片段，并统计移除的 `\` 数量

### 命令行

| 命令 | 说明 |
|------|------|
| `mpr convert --md <文件> --theme <主题>` | 规则预处理 + **已配置 API 时自动 AI 修复** |
| `mpr convert --md <文件> --fix-preview` | 仅预览规则 diff（**可不传 --theme**） |
| `mpr convert ... --no-ai-fix` | 跳过 AI 修复（仍做规则预处理） |
| `mpr convert ... --ai-fix` | 显式开启 AI 修复（默认已开启时可选） |
| `mpr convert ... --no-preprocess` | 跳过规则层 |

### 环境变量

| 变量 | 说明 |
|------|------|
| `DEEPSEEK_API_KEY` | AI 修复 / 转换自动 AI 修复所需 |
| `MARKPRESS_CONVERT_NO_AI_FIX=1` | 转换时不做 AI 修复（仍做规则预处理） |
| `MARKPRESS_CONVERT_NO_WRITE_MD=1` | 转换修复后不写回源 md |
| `MARKPRESS_MD_NO_PREPROCESS=1` | 全局跳过规则预处理 |
| `MARKPRESS_AI_VERBOSE=1` | AI 修复/转换时显示完整流式输出 |
| `MARKPRESS_INPUT` | 扫描 md 的额外目录 |

`~/.markpress/ai-config.json` 可设 `"convertAutoAiFix": false` 关闭转换自动 AI 修复。

`修复` / `检查` / `@` / 交互转换共用 `document-scan`：

- 当前工作目录、`MARKPRESS_INPUT`、`docs/`、`examples/` 下的 `.md`

## 规则预处理（① 默认、免费、即时）

**跳过范围**：仅 **fenced** 代码块（以 ` ``` ` 起止的行）内原样保留。

**未跳过（规则会处理，需注意）**：

- **缩进代码块**（行首 4 空格）：当前**会**跑规则，若含 `\` 可能被改
- **行内代码** `` `code` ``：规则按整行替换，**可能**误改反引号内的 `\`（AI 层承诺不改行内代码，规则层尚无细粒度保护）

### A. 行首结构

| 错误写法 | 正确写法 | 说明 |
|----------|----------|------|
| `\##标题` | `## 标题` | `\#` 与 `#` 后缺空格一并修复 |
| `#1212`、`##1.1 节` | `# 1212`、`## 1.1 节` | ATX 标题 `#` 后必须有空格 |
| `\- 列表项` | `- 列表项` | 无序列表（行首） |
| `1\. 第一条` | `1. 第一条` | 有序列表（行首 `\d+\.`） |
| `\> 引用` | `> 引用` | 引用块（行首） |

### B. 行内误转义（AI 导出最常见）

按顺序、**反复收敛**直到不变：

1. `\*\*` → `**`
2. `\{` `\}` → `{` `}`
3. `\_` → `_`
4. `\*` `` \` `` `\|` `\(` `\)` 等 → 去掉 `\`
5. 多层 `\\` + 符号（如 `\\\*\\\*`）→ 收敛为正常 Markdown

**行内还原字符集**：`*` `` ` `` `_` `{` `}` `[` `]` `(` `)` `#` `.` `+` `-` `!` `|`

### C. 空行 / 换行

| 问题 | 处理 |
|------|------|
| 连续 2+ 空行 | 合并为 **1** 个空行（标题/段落/列表块之间） |
| 有序列表 `1.` `2.` `3.` **之间**有空行 | **删除**（否则 HTML 间距过大、列表可能被拆） |
| 标题 / 加粗小标题与列表 **紧贴**无空行 | **插入** 1 个空行 |
| 列表块结束 → 下一段标题/正文 | **保留/插入** 1 个空行 |

实现：`normalizeBlankLines()` in `lib/markdown-preprocess.ts`

### D. 故意保留 `\` 的情况

- fenced 代码块内：不处理
- 需全文关闭规则：见上文环境变量 / `--no-preprocess`

## 规则层已知不覆盖（需 AI 或人工）

| 场景 | 说明 |
|------|------|
| Setext 标题（下划线 `===`） | 未专门处理 |
| 表格对齐、复杂嵌套列表 | 主要靠 AI |
| HTML 块 / 内联 HTML | 未专门处理 |
| 尾空格、BOM | 未处理 |
| 非 ATX 的标题风格混用 | 未处理 |
| 图片/链接 URL 内误转义 | 部分靠 B 规则，不保证 |

## AI 深度修复（② 可选）

在规则预处理之后，调用 DeepSeek（`lib/ai/markdown-fixer.ts`）。System prompt = 内置准则 + **本文件全文**。

### 必须修复

- 与 **A / B** 一致的语法问题
- 列表层级、引用嵌套明显错乱
- 表格 `\|` 误转义、列对齐明显错误

### 严格禁止

1. **不改写、不删减、不扩写**正文（含阿拉伯语、中文等多语言）
2. 不添加解释、注释、前后缀
3. **不修改** fenced 代码块内部
4. **不修改**行内代码 `` `...` `` 内部
5. 输出**不要**包在 ` ```markdown ` 里（除非原文即示例）
6. 只输出修复后的**完整 Markdown 正文**

## 正确 / 错误示例

### 正确（生成 Markdown 时请直接这样写）

```markdown
## 二级标题

**加粗文字**

1. 第一条：得分 **{win_score}**，最低 **0 分**。
2. 第二条说明。

**晋级规则**

1. 前 **{adv_num}** 支队伍晋级。
2. 同分按 **胜率** → 胜场数 → 到达该积分的先后顺序排序。
```

### 错误（不要生成；见到应修复）

```markdown
\#\# 二级标题



\*\*加粗文字\*\*



1\. 第一条



2\. 第二条
```

## marked 解析要点

| 写法 | marked 结果 |
|------|-------------|
| `#1212` | 普通段落，**不是**标题 |
| `# 1212` | `<h1>1212</h1>` |
| `\# 标题` | 段落，显示字面 `#` |
| `\*\*x\*\*` | 段落，显示字面星号，**不加粗** |
| `**x**` | 正确加粗 |

## AI 回答原则（与本页相关）

1. 用户说 **修复 / 检查 / md 不对 / 很多反斜杠** → 引导输入 **「修复」** 或 **「检查」**，优先对话指令，其次才给 `convert` 长命令
2. 用户问 **标题/加粗不生效** → 解释 `#` 后需空格、`\*\*` → `**`，建议 **修复**
3. **帮写 Markdown**：输出标准语法，**不要**给 `#`、`*`、`_` 加 `\`；占位符用 `{win_score}` 而非 `{win\_score}`
4. 修复任务**不要**输出主题 JSON（与「改风格/主题」区分）
5. **检查**不调 AI；复杂脏数据建议 **修复** 或 `convert --ai-fix`
6. 转换路径**默认已规则修复**，多数情况不必先修再转
